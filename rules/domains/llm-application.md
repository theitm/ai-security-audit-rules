# LLM Application Security Rules

Apply these rules when reviewing systems that integrate with Large Language Models (LLMs) and Generative AI.

## Prompt Injection

Check:

- User input concatenated directly into the system instructions or prompt template without clear separation.
- Lack of boundaries or delimiters between developer instructions and user-controlled content.

Fix guidance:

- Use clear, structured formats (e.g. XML tags, JSON, or unique markdown delimiters) to isolate user inputs.
- Enforce strict system instructions that warn the model against ignoring instructions within user blocks.
- Sanitize user input to remove or escape common injection delimiters (e.g. `System:`, `Human:`, triple backticks, or XML tags).

## PII and Sensitive Data Leakage

Check:

- Raw database records, environment variables, or private API keys passed directly to external LLM APIs (e.g., OpenAI, Anthropic) without filtering.
- Code forwarding user PII (emails, phone numbers, addresses, credentials) to third-party model providers.

Fix guidance:

- Run local data anonymization or redaction filters (using regex or specialized PII scrubbers) before forwarding data to external APIs.
- Use self-hosted, offline models for processing highly sensitive datasets.

## Insecure Output Handling

Check:

- Output from LLM rendered directly in the frontend using `dangerouslySetInnerHTML`, `innerHTML`, or unsafe Markdown renderers (potential XSS).
- LLM outputs passed directly into shell exec, database queries, or `eval()` blocks (potential RCE or SQLi).
- Agent loops that execute code generated by the LLM without sandbox isolation.

Fix guidance:

- Treat all LLM outputs as untrusted user input.
- Sanitize and HTML-encode LLM responses before rendering them in the UI.
- Execute any LLM-generated code inside isolated sandboxes (containers, gRPC sandboxes, VM instances) with strict network and CPU limits.

## Vietnamese notes

- Không ghép trực tiếp input của user vào chuỗi Prompt hệ thống (tránh Prompt Injection). Dùng thẻ XML, JSON hoặc dấu phân tách để cô lập.
- Lọc bỏ thông tin cá nhân nhạy cảm (PII) hoặc secret trước khi gửi sang API LLM bên thứ ba.
- Coi tất cả kết quả trả về từ LLM là dữ liệu không tin cậy. Phải sanitize trước khi render lên UI (tránh XSS) và phải chạy code sinh ra từ LLM trong sandbox cô lập (tránh RCE).
